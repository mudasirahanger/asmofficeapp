## Notes on coverage gaps

- Every backend controller that has a matching `tests/Feature/*ControllerTest.php` file was **not executed** in this pass — PHP/Composer are not installed in this sandbox and could not be installed without root (see TEST_REPORT.md). The tests exist and are reasonable in scope; they were not newly written except `test_login_is_rate_limited_after_repeated_failures` (D-2).
- Role-based UI visibility (drawer menu items, settings visibility) was reviewed by reading route/controller code, not by logging in as each role in a live app, since no live backend was available. Treat "Founder/Head only" claims above for team/settings/project-create/edit as **backend-enforced and code-reviewed**, but not independently re-verified by attempting the request as each role in this session (D-2/D-14 were verified by request/route code inspection, which is how their bugs were originally found).
- No mobile (iOS/Android native) build or test was attempted — this audit treats "mobile" as the Expo Router **web** application per the project's actual architecture (Electron wraps the web export; there's no separate native mobile app store presence discovered in this repo).
